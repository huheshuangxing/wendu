# 温度酒店 (Wendu) 项目部署指南

本文档提供了将“温度酒店”系统部署到生产服务器（或另一台局域网电脑）的详细步骤。

本项目分为**前端（Vue 3 + Vite）**和**后端（Node.js + Express + SQLite）**两部分。

## 1. 部署前准备

在开始部署之前，请确保目标服务器已安装以下环境：

- **Node.js** (推荐 v18 或更高版本)
- **npm** (随 Node.js 一同安装)
- **PM2** (用于后台持久化运行 Node.js 进程):
  ```bash
  npm install -g pm2
  ```
- **Nginx** (可选，推荐用于生产环境提供静态文件服务和反向代理)

---

## 2. 获取项目代码

通过 Git 克隆刚上传到 GitHub 的代码库：

```bash
# 替换为你的 GitHub 仓库地址
git clone https://github.com/huheshuangxing/wendu.git
cd wendu
```

---

## 3. 部署后端服务 (Node.js API + Socket.io)

后端服务负责处理数据请求、数据库读写以及 Socket 实时通讯。

```bash
# 1. 进入后端目录
cd server

# 2. 安装后端依赖
npm install

# 3. 使用 PM2 启动并守护后端服务
pm2 start server.js --name "wendu-api"

# 4. 设置 PM2 开机自启 (可选)
pm2 startup
pm2 save
```
*启动后，后端服务将默认运行在服务器的 `3000` 端口。SQLite 数据库文件 `hotel.db` 会在同目录下自动创建/读取。*

---

## 4. 部署前端项目 (Vue 3)

前端项目需要先经过构建编译，然后生成静态的 HTML/CSS/JS 文件。

```bash
# 1. 回到项目根目录
cd ..

# 2. 安装前端依赖
npm install

# 3. 构建前端生产环境代码
npm run build
```
*构建成功后，会在根目录下生成一个 `dist` 文件夹。这些就是我们要发布给用户的最终文件。*

---

## 5. 配置 Web 服务器 (以 Nginx 为例)

强烈建议在生产环境中使用 Nginx 来托管前端的 `dist` 目录，并将 API 请求转发给后端 Node.js 服务。

### Nginx 配置示例

打开 Nginx 配置文件 (通常在 `/etc/nginx/nginx.conf` 或 `/etc/nginx/conf.d/default.conf`)，添加或修改如下配置：

```nginx
server {
    listen 80;
    # 替换为你的域名或服务器 IP
    server_name your_domain_or_ip;

    # 1. 配置前端静态文件目录
    location / {
        # 替换为你的项目 dist 文件夹的绝对路径
        root /path/to/wendu/dist;
        index index.html;
        # 支持 Vue Router 的 History 模式
        try_files $uri $uri/ /index.html;
    }

    # 2. 反向代理后端的 HTTP API
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 3. 静态图片上传目录反向代理
    location /uploads/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
    }

    # 4. 反向代理 Socket.io (支持 WebSocket)
    location /socket.io/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
```

配置完成后，重启或重载 Nginx 服务：
```bash
sudo nginx -s reload
```

---

## 6. 局域网纯本地部署方案 (不使用 Nginx)

如果你只是在局域网的某台电脑（比如酒店前台电脑）上运行，不想安装 Nginx，你可以使用如下更简单的方法：

1. **后端**：按第 3 步使用 `npm start` 或 `pm2 start server.js` 保持 3000 端口运行。
2. **前端**：在项目根目录运行以下命令提供简单的预览服务：
   ```bash
   npm run preview -- --host 0.0.0.0
   ```
   该命令会将编译好的前端暴露在局域网内 (默认端口可能是 `4173` 或 `5173`)。
3. **访问**：客房平板或手机只需连接同一个 Wi-Fi，访问 `http://[前台电脑IP]:4173` 即可。

---

## 7. 注意事项

1. **防火墙设置**：无论你采用哪种部署方式，请务必确保服务器/主机的防火墙（Windows 防火墙或 Linux的 ufw/iptables）开放了相应的端口（如 Web 的 80/443 端口，或局域网预览的 5173 端口，以及后端的 3000 端口）。
2. **上传图片丢失**：项目中通过管理员后台上传的商品图片存储在 `server/uploads` 目录中。当拉取新代码或迁移服务器时，请注意**备份该目录以及 `hotel.db` 数据库文件**。
3. **网络动态识别**：前端代码已经配置为动态获取当前页面的 IP 域名（见 `src/config.ts`）。因此部署后，客户端设备通过任意合法的 IP 或域名访问页面时，都能自动正确对接后端的 3000 端口及 Socket.io 服务，无需额外修改代码。
